# Port Types

> Adapted from the [Scind specification](https://github.com/scinddev/scind). Xcind uses `XCIND_PROXY_EXPORTS` array syntax instead of YAML `exported_services`.

---

## Overview

Xcind currently supports one port exposure mechanism: **proxied** ports through Traefik via the `XCIND_PROXY_EXPORTS` configuration.

Services listed in `XCIND_PROXY_EXPORTS` are routed through the shared Traefik reverse proxy. Each export gets a hostname and Traefik routing labels are generated by `xcind-proxy-hook`.

## Proxied Ports

Traffic is routed through Traefik on the HTTP entrypoint (port configurable via `XCIND_PROXY_HTTP_PORT`, default `80`).

| Aspect | Value |
|--------|-------|
| Routing | Through Traefik via `Host()` rule |
| Hostname | Generated from URL template (e.g., `myapp-api.localhost`) |
| Protocol | HTTP (Traefik `web` entrypoint) |
| Container port | Specified in export entry or inferred from compose config |

### Export Entry Format

```
export_name[=compose_service][:port]
```

| Entry | Export Name | Compose Service | Port |
|-------|-----------|-----------------|------|
| `"api=app:3000"` | `api` | `app` | `3000` |
| `"web:8080"` | `web` | `web` | `8080` |
| `"app"` | `app` | `app` | *(inferred)* |

### Port Inference

When the port is omitted from an export entry, `xcind-proxy-hook` infers it from the compose service's port mapping:

- If the service has **exactly one** port mapping, that port's target (container port) is used
- If the service has **zero** or **multiple** port mappings, an error is reported asking for explicit port specification

Note: `yq` is required by `xcind-proxy-hook` whenever `XCIND_PROXY_EXPORTS` is configured ‚Äî for service validation, config inspection, and port inference ‚Äî not only when the port is omitted.

## Private Services

Services not listed in `XCIND_PROXY_EXPORTS` remain private ‚Äî accessible only within the application's own Docker Compose network (standard Docker Compose behavior).

In workspace mode, all services (including private ones) receive aliases on the `{workspace}-internal` network via `xcind-workspace-hook`, enabling inter-app communication.
